# Совместная разработка в Unreal Engine 5 через Git

Краткая инструкция для работы вдвоём (или в команде) с проектом **itproger_1** в Unreal Engine 5.

---

## 1. Подготовка (один раз)

### Установите Git и Git LFS

- **Git:** https://git-scm.com/download/win  
- **Git LFS** (для больших файлов .uasset, .umap): https://git-lfs.com/

После установки LFS в папке проекта выполните:

```bash
git lfs install
```

### Инициализация репозитория (если ещё не сделано)

В корне проекта (`itproger_1`):

```bash
git init
git add .
git commit -m "Initial commit: UE5 project"
```

---

## 2. Общий репозиторий (GitHub / GitLab / другой)

1. Создайте **пустой** репозиторий на GitHub (или GitLab, Bitbucket и т.п.).
2. Подключите его и отправьте код:

```bash
git remote add origin https://github.com/ВАШ_АККАУНТ/itproger_1.git
git branch -M main
git push -u origin main
```

Друг подключается так:

```bash
git clone https://github.com/ВАШ_АККАУНТ/itproger_1.git
cd itproger_1
git lfs pull
```

Затем открывает `itproger_1.uproject` в Unreal Engine 5.

---

## 3. Ежедневный workflow

### Перед началом работы

```bash
git pull
```

Если используете LFS:

```bash
git lfs pull
```

### После изменений

```bash
git status
git add .
git commit -m "Краткое описание изменений"
git push
```

Друг делает то же самое у себя и тоже пушит в общий репозиторий.

---

## 4. Важные правила для Unreal Engine

- **Не коммитить:** папки `Binaries/`, `Intermediate/`, `Saved/`, `DerivedDataCache/` — они уже в `.gitignore`.
- **Коммитить:** папку `Content/` (ассеты и карты), `Config/`, `Source/` (если есть код), `.uproject` и `.gitignore`/`.gitattributes`.
- **Один уровень — один человек:** по возможности не редактировать одну и ту же карту или один и тот же ассет одновременно, чтобы избежать конфликтов.
- **Перед pull закрыть редактор:** так меньше шансов повредить открытые ассеты при обновлении файлов.

---

## 5. Конфликты слияния

Если оба изменили один и тот же файл:

1. Git покажет конфликт при `git pull` или слиянии веток.
2. Откройте файл — в нём будут маркеры `<<<<<<<`, `=======`, `>>>>>>>`.
3. Вручную оставьте нужные фрагменты и удалите маркеры.
4. Для бинарных файлов (.uasset, .umap) конфликт обычно решают выбором одной версии:  
   `git checkout --ours путь/к/файлу.uasset` или `--theirs`, затем `git add` и завершение слияния.

Чтобы конфликтов было меньше, договаривайтесь, кто над каким уровнем/ассетом работает.

---

## 6. Ветки (по желанию)

Для фич или экспериментов:

```bash
git checkout -b feature/название-фичи
# работа...
git add .
git commit -m "Описание"
git push -u origin feature/название-фичи
```

Слияние в `main` делается через Pull Request на GitHub или вручную: `git checkout main` → `git merge feature/название-фичи` → `git push`.

---

## 7. Что уже настроено в проекте

- **`.gitignore`** — игнорируются бинарники, кэши, логи, временные файлы UE5.
- **`.gitattributes`** — для `.uasset` и `.umap` включён **Git LFS** (рекомендуется для совместной работы).

Если что-то не коммитится или, наоборот, попадает в репозиторий лишнее — проверьте и при необходимости поправьте `.gitignore`.

Удачной совместной разработки.
